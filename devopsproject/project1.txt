    ## Project Steps: AWS Key Pair Creation (xfusion-kp)

    ### Step 1: Login to AWS Console

    * Open a web browser
    * Login to **AWS Management Console**
    * Select the correct **region** (example: `us-east-1`)

    **Purpose:**
    AWS resources are region-specific.

    ---

    ### Step 2: Navigate to EC2 Service

    * Go to **Services ‚Üí EC2**
    * From the left menu, click **Key Pairs** (under *Network & Security*)

    **Purpose:**
    Key pairs are managed inside EC2.

    ---

    ### Step 3: Create a New Key Pair

    * Click **Create key pair**
    * Enter details:

    * **Key pair name:** `xfusion-kp`
    * **Key pair type:** `RSA`
    * **Private key file format:** `.pem`

    **Purpose:**
    RSA key pairs are used for secure SSH authentication.

    ---

    ### Step 4: Download the Key File

    * AWS will automatically download:

    ```
    xfusion-kp.pem
    ```
    * Save it in a secure location

    ‚ö†Ô∏è **Important:**
    If you lose this file, it **cannot be downloaded again**.

    ---

    ### Step 5: Set Correct Permissions (Linux/Mac)

    ```bash
    chmod 400 xfusion-kp.pem
    ```

    **Purpose:**
    SSH requires strict file permissions for private keys.

    ---

    ### Step 6: Connect to EC2 Instance Using the Key

    ```bash
    ssh -i xfusion-kp.pem ubuntu@<ec2-public-dns>
    ```

    **Purpose:**
    Verifies that the key pair works correctly.

    ---

    ### Step 7: Verify Successful Login

    * If login succeeds, the key pair is working
    * No password will be required

    ---
